MySQL数据类型

  • 数值型

    tinyint smallint mediumint int bigint float double decimal

    补充

    1. unsigned 表示 无符号

    2. float(M,D) 浮点型

    3. decimal(M,D) 定点型 比 float 更加精准

    4. M：精度（总位数） D：标度（小数位）

  • 字符型

    1. char(32) ：最长32，定长， hello（5个字符，剩下的 27 个字符就会被浪费，但是效率高）
    2. varchar(32)： 最长不能超过32 变长 hello(5个字符，会计算出5 个字符的空间，剩余的 空间不会给出，不浪费空间，但是效率低)
    3. text ：文本 可以存一篇文章

  • 特殊类型NULL

    1. NULL 不是假 也不是真，而是空
    2. NULL 的判断只能用 is null , is not null
    3. NULL 影响查询速度，一般避免使用NULL

存储引擎和字符集

  • 存储引擎

    数据库对同样的数据有不同的管理模式，怎么实现的，就是根据 存储引擎实现的

    • 不同引擎之间的 区别
    • 根据场景选择 引擎
      1. 文章新闻 等安全性要求不高的时候 ，使用 myisam
      2. 订单 资金 账单 火车票等 对安全性要求高的 选 innodb
      3. 对于 临时表中转表，使用 memory ，速度快

    MySQL 在5.6 版本以上，默认是使用innodb

  • 字符集

字符集是一套符号和编码的规则，不管哪个 数据库 都存在 字符集选择的问题

如果在创建之初 没有设置好，那么后期 再改，会有一定的风险

建议在建设之初就设置好

  1. 查看所有字符集

    show variables like 'character_%'

  1. 客户端与服务端 之间的字符集编码的转换（中间存在一个转换器）

    • character_set_client => utf8 （客户端告诉转换器发过来的是utf8格式的编码）

    • character_set_connection => utf8 （将客户端传过来的数据 用utf8 来转换）

    • character_set_results => utf8 （告诉客户端 结果数据的编码 是utf8）

      以上三个字符集可以使用 set names utf8 来统一进行设置
